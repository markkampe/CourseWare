Most of the labs in this course involve writing and testing C code on a Linux system. Most of your notebooks are likely running MacOS or Windows, which will not be able to run most of these projects. Fortunately we have a departmental Linux server (wilkes.cs.hmc.edu) set up for your use. You will need to:

  • If you do not already have one, get a cs.hmc.edu account, confirm that you can log in with it.
    It would be best if you got this under control before showing up for the first lab session.

  • From your notebook (or a lab machine), open an ssh (putty on Windows systems) session to wilkes.cs.hmc.edu and confirm that you are able to log in.
    Create a new sub-directory for your projects in this course: 
     wilkes% cd			# make sure you are in your $HOME directory
 wilkes% mkdir CS105_projects	# you can choose any name you like
 wilkes%
  • If you do not already have it, download and install Visual Studio Code (from Microsoft) on your personal machine, and add the remote ssh connection extension.
    Open a remote Visual Studio connection to wilkes.cs.hmc.edu, and change to the new sub-directory you created for these projects.
    In the remote session, start a terminal session so you can type shell commands to compile and run your programs.

  • You may find that Visual Studio regularly asks you to re-enter your password to establish new connections. If this becomes bothersome, you can (there are numerous tutorials on the internet):
    • If you do not already have one, use the ssh-keygen command to create a new RSA public/private key pair on your personal notebook (do not do this on a lab machine).
    • install a copy of the public key (id_rsa.pub,with rw------- protection) in the .ssh sub-directory of your $(HOME) directory on Wilkes.
    Once you have done this, ssh and Visual Studio should be able to automatically log in to Wilkes (from your personal notebook) without your needing to enter your password.

There are (at least) two ways to get a lab-starter onto Wilkes:

  • download it onto your local machine, and then drag that file into the (remote session) Visual Studio Explorer window.

  • in a remote terminal session, use the Linux wget command to directly download the lab-starter onto Wilkes. 
    wilkes% wget https://cs.hmc.edu/courses/cs105/projects/bits-handout.tar
wilkes%
    Note: you can copy the URL for the hand-out by hovering over the link (e.g. in the course schedule) and doing a right-click "copy link location", and then paste it after your wilkes wget command.
Either way you will then have to use the Linux tar command to extract the files: 
	wilkes% tar xvf bits-handout.tar
	wilkes%
In most cases, this will create a new sub-directory just for that project.
